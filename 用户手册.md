# cuteFC-web 用户手册

## 1. 文档说明

本文档面向 `cuteFC-web` 部署包的最终用户，说明本地 Web 应用的安装、配置、启动与使用方法。

需要特别说明：

- `cuteFC` 本身既可以作为**命令行软件**使用，也可以作为**Web 本地应用**部署使用。
- 当前这份 `cuteFC-web` 发布包提供的是 **本地 Web 应用部署与启动方式**。
- 本手册主要围绕当前部署包中已经提供并实际接入的前端页面与后端能力展开说明。

---

## 2. 产品概览

`cuteFC-web` 是一个面向结构变异（SV）分析场景的本地 Web 平台，围绕 `cuteFC` 构建，提供以下能力：

1. **项目管理**
   - 创建项目（Project / Workspace）
   - 在项目中登记样本
   - 为样本或项目登记输入文件路径

2. **cuteFC 分析任务管理**
   - 创建 cuteFC 任务
   - 配置平台类型、线程数和算法参数
   - 启动任务、查看状态、查看日志

3. **结果浏览与导出**
   - 浏览任务输出的结构变异结果
   - 按染色体、区间、SV 类型、基因型、长度、支持读段数筛选
   - 查看单条变异详情
   - 在满足条件时使用 IGV.js 浏览位点上下文
   - 导出 CSV / TSV / VCF

4. **群体图谱浏览**
   - 导入并管理群体 VCF 数据集
   - 浏览队列级结构变异概况
   - 按基因、样本、SV 类型、区间、频率等条件检索

5. **Demo 演示数据**
   - 默认启动时自动导入一套演示项目、样本、任务结果和群体图谱数据
   - 便于首次体验与功能验证

---

## 3. 部署包包含内容

当前部署包保留了运行本地 Web 应用所需的最小运行时组件，主要包括：

- `backend/app/`：FastAPI 后端服务
- `frontend/src/` 与 `frontend/public/`：React 前端界面
- `cuteFC/`：随包提供的 `cuteFC` 运行时
- `local_vendor/cigar.py`：本地依赖
- `data/demo/`：内置演示数据
- `start.sh`：一键启动脚本

## 4. 系统要求

根据启动脚本和前端配置，建议环境如下：

- **操作系统**：Linux 或 macOS Shell 环境
- **Python**：3.11 及以上
- **Node.js**：20 及以上
- **npm**：可用

说明：

- 启动脚本会自动创建 `.venv` 虚拟环境。
- 后端默认使用 **SQLite** 数据库，数据库文件默认位于 `./data/app.db`。
- 前端默认通过 Vite 开发服务器提供界面。

## 5. 安装与启动

### 5.1 首次启动

在项目根目录执行：

```bash
chmod +x start.sh
./start.sh
```

脚本会自动完成以下动作：

1. 检查 Python 与 npm 是否可用
2. 创建或复用 `.venv`
3. 安装后端依赖
4. 以可编辑方式安装随包提供的 `cuteFC`
5. 安装前端依赖
6. 启动 FastAPI 后端
7. 等待后端健康检查通过
8. 默认自动注入内置 demo 数据
9. 启动前端页面

启动成功后，终端会输出：

- 前端地址：`http://127.0.0.1:5173`
- 后端地址：`http://127.0.0.1:8000`

停止服务：

- 在当前终端按 `Ctrl + C`

### 5.2 再次启动

如果依赖已经安装完成，可跳过安装阶段：

```bash
SKIP_INSTALL=1 ./start.sh
```

### 5.3 不自动导入 Demo 数据

如果你希望以“空系统”方式启动：

```bash
BOOTSTRAP_DEMO=0 ./start.sh
```

### 5.4 自定义端口

```bash
BACKEND_PORT=18000 FRONTEND_PORT=15173 ./start.sh
```

如果默认端口被占用，脚本会直接报错并提示修改端口。

## 6. 配置说明

本部署包主要通过**环境变量**进行配置。

### 6.1 启动脚本相关环境变量

#### `SKIP_INSTALL`
- 默认值：`0`
- 作用：是否跳过依赖安装
- 常见用法：

```bash
SKIP_INSTALL=1 ./start.sh
```

#### `BOOTSTRAP_DEMO`
- 默认值：`1`
- 作用：是否在启动后自动导入演示数据
- 常见用法：

```bash
BOOTSTRAP_DEMO=0 ./start.sh
```

#### `BACKEND_PORT`
- 默认值：`8000`
- 作用：后端服务端口

#### `FRONTEND_PORT`
- 默认值：`5173`
- 作用：前端服务端口

#### `BACKEND_HOST`
- 默认值：`127.0.0.1`
- 作用：终端输出中显示的后端访问地址

#### `FRONTEND_HOST`
- 默认值：`127.0.0.1`
- 作用：终端输出中显示的前端访问地址

### 6.2 后端配置项

后端配置位于 `backend/app/config.py`，当前支持：

#### `DATABASE_URL`
- 默认值：`sqlite:///./data/app.db`
- 作用：数据库连接地址
- 默认使用 SQLite，本地部署无需额外数据库服务

#### `DATA_ROOT`
- 默认值：`./data`
- 作用：后端数据根目录
- 用于数据库文件、任务输出目录、日志目录等

#### `CORS_ORIGINS`
- 作用：允许访问后端的前端来源
- 启动脚本会自动设置为对应前端端口的 `localhost` 和 `127.0.0.1`

### 6.3 前端配置项

前端通过 `VITE_API_BASE_URL` 指向后端：

- 默认由启动脚本设置为：`http://127.0.0.1:${BACKEND_PORT}`

一般情况下无需手动修改。

## 7. 数据与目录说明

### 7.1 数据库

默认数据库文件：

```text
./data/app.db
```

用于保存：

- 项目
- 样本
- 已登记文件
- 分析任务
- 结果变异记录
- 群体数据集及其汇总结果

### 7.2 任务运行目录

任务运行后，系统会在数据目录下生成：

- 任务日志目录
- 任务输出目录

后端会记录每个任务的：

- `log_path`
- `output_path`

### 7.3 输入文件登记机制

本系统**不会上传文件内容到数据库**，而是登记文件的**绝对路径**。因此：

- BAM / CRAM
- FASTA
- 目标 VCF
- 结果 VCF

都需要以本机可访问的路径形式提供。

这意味着：

1. Web 页面和后端必须运行在能访问这些文件的同一台机器上
2. 路径必须真实存在
3. 若路径不存在，任务运行时会失败

## 9. 页面与功能说明

本地 Web 应用的主要导航包括：

- `Dashboard`
- `Platform`
- `Pipeline`
- `Genetic Map`
- `Showcase`

### 9.1 首页 Landing Page

根路径 `/` 是产品展示页，主要用于：

- 展示 cuteFC 的总体定位
- 展示方法学与示意图片
- 提供进入应用的入口

这部分更偏介绍与展示，不承担数据操作功能。

### 9.2 Dashboard：总览页

路径：`/app`

Dashboard 会从后端读取总览数据，展示：

- 系统标题与副标题
- 统计卡片
- 结构变异类型分布
- 热点位点（Hotspots）
- 最近任务活动（Recent Tasks）

适用场景：

- 快速了解当前系统中已有多少任务和变异结果
- 查看最近任务状态
- 从总览页快速跳转到新建分析或群体图谱页面

### 9.3 Platform：项目管理

路径：`/app/platform`

主要功能：

- 查看已有项目列表
- 创建新项目
- 进入单个项目详情页

项目用于组织：

- 一个研究主题
- 一组样本
- 相关输入文件
- 该项目下产生的分析任务

#### 创建项目

需要填写：

- 项目名称
- 项目说明

创建后会出现在项目列表中。

### 9.4 Project Detail：项目详情

路径：`/app/platform/:projectId`

这是日常使用中的核心页面之一，主要包括三部分：

#### 1）样本列表（Biological Cohort）
显示当前项目下已登记的样本，包括：

- 样本名
- 测序平台类型（ONT / HiFi / CLR）
- 备注
- 已登记到该样本的文件

#### 2）样本新增（Add Sample）
可新增样本，字段包括：

- `sample_name`：样本名
- `platform_type`：测序平台类型
- `remark`：备注

当前界面支持的平台类型：

- ONT
- HiFi
- CLR

#### 3）文件登记（Register Files）
用于登记任务运行需要的输入文件路径。

可登记的文件类型：

- `bam`：BAM / CRAM
- `reference_fasta`：参考基因组 FASTA
- `target_vcf`：目标位点 VCF

文件可以登记到：

- 某个具体样本
- 整个项目共享资源（`Shared Project Resource`）

系统在创建任务时会优先读取：

1. 当前样本专属文件
2. 若样本专属文件不存在，则回退到项目级共享文件

这对多个样本共用同一参考 FASTA 或目标 VCF 的场景很有用。

#### 4）项目任务摘要（Pipelines）
页面右侧会展示该项目下任务的简要信息，并提供“Configure Pipeline”入口跳转到任务创建页。

## 10. cuteFC 分析任务使用说明

### 10.1 任务列表页 Pipeline

路径：`/app/pipeline`

功能包括：

- 查看全部任务
- 按任务名搜索
- 按状态筛选
- 按项目筛选
- 按平台类型筛选
- 查看任务统计摘要

任务状态包括：

- `pending`：待运行
- `running`：运行中
- `succeeded`：已完成
- `failed`：失败

任务列表适合用于：

- 管理多个分析任务
- 查找失败任务并重试
- 进入任务详情或结果页

### 10.2 新建任务

路径：`/app/pipeline/new`

也可以从项目详情页进入，并自动带入项目 ID。

#### 创建任务时需要配置的内容

##### 1）项目与样本
- 选择项目
- 选择样本

系统会根据样本自动推断：

- 平台类型
- 默认任务名
- 该样本可用的 BAM / FASTA / 目标 VCF 路径

##### 2）平台类型
支持：

- ONT
- HiFi
- CLR

不同平台会带入不同的默认参数预设。

##### 3）线程数
- 字段：`threads`
- 默认值：`8`

##### 4）算法参数
前端当前暴露了三个参数：

- `min_support`
- `min_size`
- `min_mapq`

不同平台的默认值为：

| 平台 | min_support | min_size | min_mapq |
|---|---:|---:|---:|
| ONT | 3 | 50 | 20 |
| HiFi | 2 | 50 | 20 |
| CLR | 4 | 50 | 10 |

##### 5）数据路径
需要提供：

- `input_bam`
- `reference_fasta`
- `target_vcf`
- `output_vcf`
- `work_dir`

其中：

- `input_bam`、`reference_fasta`、`target_vcf` 一般会尝试从已登记文件中自动带出
- `output_vcf` 与 `work_dir` 会自动生成默认路径，但你也可以手动修改

#### 任务创建后的状态
创建成功后，任务会先进入：

- `pending`

此时只是保存了任务配置，**尚未真正运行**。

### 10.3 运行任务

进入任务详情页后，可点击：

- `运行任务`
- 失败后可点击 `重新运行`

后端在运行任务时会做以下事情：

1. 校验输入文件是否存在
   - `input_bam`
   - `reference_fasta`
   - `target_vcf`

2. 自动创建工作目录与输出目录
3. 根据平台和参数生成 `cuteFC` 命令
4. 调用随包提供的 `cuteFC` 执行分析
5. 写入日志文件
6. 若成功，则自动解析输出 VCF 并入库

#### 实际调用方式
后端会把任务组装为命令行运行，因此本地 Web 应用本质上是：

- 负责组织参数
- 调度 `cuteFC` 命令
- 存储结果并提供可视化

这也再次说明：

- `cuteFC` 既可命令行单独运行
- 也可由当前 Web 本地应用作为后端分析引擎来调用

## 12. 结果浏览与解释

路径：`/app/results/:taskId`

当任务执行成功且结果 VCF 被解析入库后，可在结果页浏览。

### 12.1 结果页主要内容

结果页包括以下模块：

#### 1）结果总览
展示：

- 总 SV 数量
- 中位 SV 长度
- 最大 SV 长度
- 主导 SV 类型
- SV 类型分布
- 基因型分布
- 染色体分布
- 长度分桶分布
- 支持读段数分桶分布

#### 2）结果筛选器
支持按以下条件筛选：

- 染色体 `chrom`
- 起始位置 `start`
- 终止位置 `end`
- SV 类型 `sv_type`
- 基因型 `genotype`
- 最小 / 最大长度
- 最小 / 最大支持读段数
- 排序字段
- 排序顺序
- 每页显示条数

#### 3）结果表格
展示单条结构变异记录，包括：

- 染色体
- 起始位置
- 终止位置
- SV 类型
- SV 长度
- 基因型
- 支持读段数

支持：

- 按列排序
- 翻页
- 点击某条记录查看详情

#### 4）结果导出
支持导出当前筛选结果为：

- CSV
- TSV
- 标准 VCF

导出内容基于当前筛选条件生成，而不是固定导出全部数据。

### 12.2 单条变异详情

点击结果表中的某条变异后，可查看：

- 所属任务与样本
- SV 类型
- 基因型
- 长度
- 支持读段数
- 染色体坐标
- locus 文本表示
- 相关文件路径
  - FASTA
  - BAM
  - 输出 VCF
  - 目标 VCF

### 12.3 IGV 位点浏览

若相关文件与索引满足条件，系统会尝试在页面中嵌入 IGV.js 浏览器。

#### 需要满足的条件
至少需要：

- 参考 FASTA 存在
- FASTA 对应 `.fai` 索引可用
- 输出 VCF 存在

若希望看到 BAM 比对轨道，还需要：

- 输入 BAM 存在
- BAM 对应 `.bai` 索引存在

若目标 VCF 存在，也会作为额外变异轨道加载。

#### 索引文件说明
- 参考 FASTA：若 `.fai` 不存在，后端会尝试自动生成
- BAM：若缺少 `.bai`，则无法显示 alignment 轨道
- 目标 VCF：若缺少 `.tbi`，当前详情中会被标记为可用性不足

如果 IGV 无法初始化，界面会提示原因，例如：

- 参考 FASTA 不存在
- 缺少 `.fai`
- 输出 VCF 不存在
- BAM 缺少 `.bai`

## 13. 群体图谱功能

### 13.1 Genetic Map：群体图谱浏览

路径：`/app/genetic-map`

这是当前群体检索的主要工作页面，已和后端真实接口打通。

页面功能包括：

1. 自动加载已导入成功的群体数据集
2. 展示队列概况
   - 数据集 ID
   - 样本总数
   - 位点总数
   - hotspot 数量
   - rare 数量
3. 提供检索栏
4. 提供快捷 SV 类型过滤（DEL / INS / DUP / INV / BND）
5. 展示按染色体分组的变异分布视图
6. 展示变异目录表格
7. 支持点击查看单条队列变异记录

#### 支持的检索条件
后端当前支持：

- `gene_name`
- `sample_name`
- `sv_type`
- `chrom`
- `start`
- `end`
- `frequency_min`
- `frequency_max`

#### 群体数据中每条记录包含的信息
- variant_key
- chrom
- pos
- end
- sv_type
- sample_count
- frequency
- sample_list
- gene_name

这适合用于：

- 浏览热点结构变异位点
- 查找低频变异
- 按样本或基因回溯变异
- 初步进行群体层面的 SV 检索

### 13.2 Cohort Search 页面说明

路径：`/app/genetic-map/search`

前端提供了一个单独的高级检索页面外观，但从当前实现看，该页面仍以界面展示为主，并**没有真正接入后端返回结果列表**。

因此当前建议：

- **实际使用群体检索时，以 `/app/genetic-map` 页面为主**。

## 16. 典型使用流程

### 16.1 快速体验流程（推荐首次使用）

1. 运行 `./start.sh`
2. 打开前端地址
3. 在 Dashboard 查看总览
4. 进入 Platform 查看演示项目
5. 进入 Pipeline 查看演示任务
6. 打开 Results 查看演示结果
7. 打开 Genetic Map 浏览队列级图谱

### 16.2 从零开始创建自己的分析流程

1. 启动系统（可设置 `BOOTSTRAP_DEMO=0`）
2. 进入 `Platform`
3. 创建项目
4. 进入项目详情页
5. 新增一个或多个样本
6. 为样本或项目登记：
   - BAM / CRAM
   - Reference FASTA
   - Target VCF
7. 进入 `Pipeline` 新建任务
8. 检查自动带入的路径与参数
9. 创建任务
10. 进入任务详情页并点击运行
11. 运行成功后进入结果页查看与导出

### 16.3 使用群体图谱流程

1. 先确保系统中已有成功导入的数据集
2. 打开 `Genetic Map`
3. 在检索栏输入基因、样本、SV 类型或区间
4. 查看筛选后的队列级变异列表
5. 结合频率与样本列表判断感兴趣位点

## 17. 输入文件准备建议

为了顺利运行任务，建议准备以下文件：

### 17.1 BAM / CRAM
- 需要是目标样本的比对结果文件
- 若希望在 IGV 中显示读段轨道，建议同时准备 `.bai` 索引

### 17.2 Reference FASTA
- 需要是与 BAM / 目标 VCF 相匹配的参考基因组
- 若无 `.fai`，后端可能会自动生成，但仍建议预先准备好

### 17.3 Target VCF
- 用于指定待分析目标位点
- 若希望作为轨道在 IGV 中显示，建议同时准备索引文件

### 17.4 输出路径
- `output_vcf` 必须写入到可写目录
- `work_dir` 建议为任务独立目录，避免不同任务相互覆盖

## 18. 常见问题与排查

### 18.1 启动时报缺少 Python 或 npm
请先安装：

- Python 3.11+
- Node.js 20+
- npm

### 18.2 提示端口已占用
修改端口重新启动：

```bash
BACKEND_PORT=18000 FRONTEND_PORT=15173 ./start.sh
```

### 18.3 页面能打开，但数据加载失败
优先检查：

1. 后端是否已成功启动
2. 前后端端口是否一致
3. 终端中是否有 FastAPI 报错
4. `VITE_API_BASE_URL` 是否正确

### 18.4 任务创建成功但运行失败
常见原因：

- `input_bam` 不存在
- `reference_fasta` 不存在
- `target_vcf` 不存在
- 输出目录不可写
- 参数不合理

建议：

- 到任务详情页查看日志与错误信息
- 先确认路径真实存在，再重试

### 18.5 有结果记录，但 IGV 不显示
优先检查：

- FASTA 是否存在
- FASTA 是否有 `.fai`
- BAM 是否有 `.bai`
- 输出 VCF 是否存在

### 18.6 为什么登记的是“路径”而不是上传文件？
因为当前部署包的设计是：

- 本地运行
- 由后端直接访问本机文件系统
- 避免在 Web 层重复存储大型基因组文件

因此用户需要自己保证这些路径对后端可访问。

## 19. 当前实现边界说明

根据当前前后端实现，以下几点需要用户知晓：

1. **这是本地 Web 应用部署包，不是云端托管版**
   - 数据和文件都保留在本地环境中

2. **文件登记是路径登记，不是文件上传**
   - 系统依赖本机文件系统访问

3. **群体图谱高级搜索的主入口是 Genetic Map 页面**
   - `/app/genetic-map/search` 当前更像预留界面

4. **任务是手动触发执行**
   - 创建任务后不会自动开始运行
   - 需要进入任务详情页点击运行

5. **IGV 显示依赖索引文件完整性**
   - 特别是 `.fai` 与 `.bai`

## 20. 总结

`cuteFC-web` 提供了一套围绕 `cuteFC` 的本地 Web 工作台，适合用于：

- 组织项目与样本
- 管理 BAM / FASTA / VCF 等输入资源
- 创建并运行 cuteFC 结构变异分析任务
- 浏览、筛选和导出结果
- 查看群体级结构变异图谱

如果你的目标是：

- 直接在终端中调用 `cuteFC`，可以将其作为命令行软件单独使用；
- 如果希望获得更完整的项目管理、结果浏览和可视化体验，则可以使用当前部署包提供的本地 Web 应用方式。
